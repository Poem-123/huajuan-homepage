# 花卷的自我介绍网页

这是一个静态个人主页项目。打开 `index.html` 就能看到页面；发布到 GitHub Pages 后，浏览器会先读取 HTML，再加载 CSS 和 JavaScript。

## 建议阅读顺序

1. `index.html`
   页面内容和结构。可以把它理解成网页的“骨架”：标题、导航、简介、技能、作品、联系方式都写在这里。

2. `style.css`
   页面外观。可以把它理解成网页的“衣服”：颜色、字体、间距、卡片、深色主题、手机端布局都在这里。

3. `script.js`
   页面交互。可以把它理解成网页的“动作”：明暗主题切换、展开/收起、返回顶部都在这里。

4. `微信图片_20260624111924_2_2.jpg`
   页面使用的头像图片。`index.html` 通过 `<img>` 标签引用它。

5. `.gitignore`
   告诉 Git 哪些文件不要上传，例如 macOS 的 `.DS_Store` 和本地 `.command` 脚本。

## 这次改了 3 处

1. 修正项目和联系区链接：把 `#`、占位 GitHub、占位 LinkedIn 换成真实 GitHub 仓库和公开页面链接。
2. 增强键盘可访问性：给导航、按钮、项目链接、联系方式链接增加清晰的 `focus-visible` 焦点样式。
3. 加强脚本稳定性：`script.js` 现在会先确认元素存在再绑定事件，未来删改某个按钮时不容易让整段脚本报错。

## 零基础看代码的小提示

- HTML 里 `id="intro"` 这样的 id 是“锚点名字”，导航里的 `href="#intro"` 会跳到它。
- CSS 里 `.site-header` 这样的写法是在给 `class="site-header"` 的元素设置样式。
- JavaScript 里 `addEventListener("click", ...)` 表示“当用户点击时，执行后面的代码”。
